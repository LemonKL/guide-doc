<h1>一、定义</h1>
<p>一棵二叉查找树是一棵二叉树，每个节点都含有一个Comparable的键（以及对应的值）。</p>
<p>每个节点的键都大于左子树中任意节点的键而小于右子树中任意节点的键。</p>
<p>&nbsp;</p>
<p>每个节点都有两个链接，左链接、右链接，分别指向自己的左子节点和右子节点，链接也可以指向null。</p>
<p>尽管链接指向的是节点，可以将每个链接看做指向了另一棵二叉树。这个思路能帮助理解二叉查找树的递归方法。</p>
<p>&nbsp;</p>

<h2>3、查找实现（search）</h2>
<p>思路：</p>
<p>A、如果二叉查找树为空，查找失败（search miss），返回null；</p>
<p>B、如果根节点的键等于要查找的键，返回根节点的值（search hit）。</p>
<p>C、否则，继续在<strong>相应的子树</strong>中查找。如果要查找的键小于根节点的键，在左子树中查找；如果要查找的键大于根节点的键，在右子树中查找。</p>
<p>D、重复ABC步骤，直至search miss或者search hit。</p>
<p>&nbsp;</p>
<p>递归实现：</p>

     * Returns the value associated with the given key.
     * 
     * @param     key the key
     * @return     the value associated with the given key if the key is in the symbol table
     *             and {@code null} if the key is not in the symbol table
     * @throws    IllegalArgumentException if {@code key} is {@code null}

        if(x == null)
            return null;//serach miss
        
        int cmp = key.compareTo(x.key);

            return get(x.left, key);
        else if(cmp &gt; 0)
            return get(x.right, key);
        else
            return x.val;//serach hit
        
    }</pre>

<h2>4、插入实现</h2>
<p>思路：</p>
<p>A、如果二叉查找树是空的，生成一个新节点，并返回该节点，相当于插入新节点后的二叉树根节点。</p>
<p>B、如果根节点键和要插入的键相等，更新根节点的值。</p>
<p>C、如果要插入的键小于根节点的键，在左子树插入，并将根节点的左链接指向插入后的左子树。</p>
<p>D、如果要插入的键小于根节点的键，在右子树插入，并将根节点的右链接指向插入后的右子树。</p>
<p>E、更新根节点的size，并返回根节点，作为插入新节点后的二叉树根节点。</p>
<p>F、重复ABCD，直至插入或者更新成功。</p>
<p>&nbsp;</p>
<p>递归实现：</p>

<pre>　　/**
     * Inserts the specified key-value pair into the symbol table, overwriting the old 
     * value with the new value if the symbol table already contains the specified key.
     * Deletes the specified key (and its associated value) from this symbol table
     * if the specified value is {@code null}.
     * 
     * @param key the key
     * @param val the value
     * @throws IllegalArgumentException if {@code key} is {@code null}
     */
    public void put(Key key, Value val) {
        
        if(key == null)
            throw new IllegalArgumentException("key is null");
        
        if(val == null) {
            delete(key);
            return;
        }
        
        root = put(root, key, val);
    }
    
    private Node put(Node x, Key key, Value val) {
        
        if(x == null) 
            return new Node(key, val, 1);
        
        int cmp = key.compareTo(x.key);
        
        if(cmp &lt; 0)
            x.left = put(x.left, key, val);
        else if(cmp &gt; 0) 
            x.right = put(x.right, key, val);
        else
            x.val = val;
        
        x.size = size(x.left) + size(x.right) + 1;
        
        return x;
        
    }</pre>

<pre>/**
     * Inserts the specified key-value pair into the symbol table, overwriting the old 
     * value with the new value if the symbol table already contains the specified key.
     * Deletes the specified key (and its associated value) from this symbol table
     * if the specified value is {@code null}.
     * 
     * @param key the key
     * @param val the value
     * @throws IllegalArgumentException if {@code key} is {@code null}
     */
    public void put(Key key, Value val) {
        
        if(key == null)
            throw new IllegalArgumentException("key is null");
        
        if(val == null) {
            delete(key);
            return;
        }
        
        if(root == null) {
            root = new Node(key, val);
            return;
        }
        
        boolean alreadyin = contains(key);//see if it needs to update the counts
        Node parent = null;
        Node cur = root;
        
        while(cur != null) {
            parent = cur;
            cur.size = alreadyin ? cur.size : cur.size + 1;//change size of cur
            
            int cmp = key.compareTo(cur.key);
            if(cmp &lt; 0)
                cur = cur.left;
            else if(cmp &gt; 0)
                cur = cur.right;
            else {
                cur.val = val;
                return;
            }
        }
        
        if(key.compareTo(parent.key) &lt; 0)
            parent.left = new Node(key, val);
        else
            parent.right = new Node(key, val);
        
    }</pre>

<h2>6、floor和ceiling实现</h2>
<p>floor是求the largest key in the BST less than or equal to&nbsp;<em>key</em></p>
<p>思路：</p>
<p>A、如果根节点是null，则直接返回null。</p>
<p>B、如果根节点键值大于<em>key</em>，则继续floor(key)在左子树中，所以继续在左子树中查找。</p>
<p>C、如果根节点键值刚好等于<em>key</em>，则根节点的键值即为floor(key)，直接返回该键。</p>
<p>D、如果根节点键值小于<em>key</em>，那么根节点的key有可能就是floor(key)，<strong>只要</strong>右子树中不存在节点的key小于等于输入的<em>key。</em></p>
<p>E、重复ABCD</p>
<p>&nbsp;</p>
<p>递归实现：</p>

<p>A、用t保存即将删除的节点x。</p>
<p>B、将x指向右子树的最小键节点，也就是继承者。</p>

<p>B、还要处理一个<strong>特殊情况</strong>，就是<strong>根节点</strong>就是要删除的节点。</p>
<p>C、还要多调用一次get，用于判断要删除的节点是否在二叉搜索树中。</p>
<p>D、非递归实现更新size跟递归实现的顺序相反，要<strong>边搜索边更改size</strong>。</p>

<h1>三、性能分析</h1>
<p>假设键的顺序是随机的，也就是插入的顺序是随机的。</p>
<p>这个的分析其实和快排是差不多的。</p>
<p><strong>结论1：</strong>在N个随机键构造的二叉查找树中，<strong>查找命中</strong>平均需要的比较次数为~2lnN ~1.39log<sub>2</sub>N。</p>
<p>证明：</p>
<p>定义<strong>内部路径长度</strong>为所有节点的<strong>深度</strong>的和。</p>
<p>令I<sub>N</sub>为N个随机排序的不同键<strong>构造</strong>而成的二叉查找树的内部路径长度，其中I<sub>0</sub>=I<sub>1</sub>=0。</p>
<p>则I<sub>N</sub>&nbsp;= （<strong>N-1</strong>） + （I<sub>0</sub>+I<sub>N-1</sub>）/N + （I<sub>1</sub>+C<sub>N-2</sub>）/N + &hellip;&hellip; + （I<sub>N-1</sub>+I<sub>0</sub>）/N。</p>
<p>根节点使两个子树所有节点的深度都加1，也就是<strong>N-1</strong>。</p>
<p>这个的分析跟快排的分析差不多，I<sub>N</sub>~2NlnN。</p>
<p>平均比较次数为：1+I<sub>N</sub>/N~2lnN。</p>
<p>&nbsp;</p>
<p><strong>结论2：</strong>在N个随机键构造的二叉查找树中，<strong>插入和查找失败</strong>平均需要的比较次数为~2lnN ~1.39log<sub>2</sub>N</p>
<p>证明：</p>
<p>定义<strong>外部路径长度</strong>为根节点到所有null节点的所有路径的节点总和。</p>
<p>令E<sub>N</sub>为N个随机排序的不同键<strong>构造</strong>而成的二叉查找树的外部路径长度，其中E<sub>0</sub>=0，E<sub>1</sub>=2。</p>
<p>则E<sub>N</sub>&nbsp;= （N+1） + （E<sub>0</sub>+E<sub>N-1</sub>）/N + （E<sub>1</sub>+E<sub>N-2</sub>）/N + &hellip;&hellip; + （E<sub>N-1</sub>+E<sub>0</sub>）/N。</p>
<p>其中<strong>N+1</strong>为经过根节点的总路径的总和为<strong>N+1</strong>。这里还需要证明有N个节点的二叉树，其null链接为N+1。所以总路径数为N+1。</p>
<p>N个节点，总共有2N个链接，其中除了根节点外，每个节点都有一个链接指向该节点，故总共有N-1个链接指向节点，剩下2N-(N-1)=N+1个链接指向null。</p>
<p>E<sub>N</sub>的分析和I<sub>N</sub>差不多。</p>
<p>&nbsp;</p>
<p><strong>结论3：</strong>E<sub>N</sub>=I<sub>N</sub>+2N</p>
<p>用归纳法，即可证明。略。</p>
<p>&nbsp;</p>
<p><strong>结论4：插入和查找失败平均比查找成功多一次比较。</strong></p>
<p>证明：</p>
<p>插入和查找失败平均比较次数为E<sub>N</sub>/N，查找成功平均比较次数为I<sub>N</sub>/N+1。</p>
<p>EN/N-(I<sub>N</sub>/N+1)=(EN-IN)/N - 1= 2N/N - 1 = 1。</p>
<p>&nbsp;</p>
<p><strong>结论5：</strong>在一棵二叉查找树中，所有操作在最坏情况下所需的时间都和树的<strong>高度</strong>成正比。</p>
<p>树的高度将会比平均内部路径长度要大。对于足够大的N，高度趋近于<strong>2.99logN</strong>。</p>
<p>&nbsp;</p>
<p>但是构造树的键<strong>不是随机</strong>的话，最坏情况将会变得不可接受。比如用例按照顺序或者逆序插入符号表。</p>
<p>这种情况还是有可能出现的，因为用例控制着插入和查找等操作的顺序。</p>
<p><strong>平衡二叉查找树（红黑树）</strong>可以解决这个问题。</p>
