<h3><a name="a1"></a><strong>R-B Tree简介</strong></h3>
<p>&nbsp; &nbsp; R-B Tree，全称是Red-Black Tree，又称为&ldquo;红黑树&rdquo;，它一种特殊的二叉查找树。红黑树的每个节点上都有存储位表示节点的颜色，可以是红(Red)或黑(Black)。</p>
<p><strong>红黑树的特性</strong>:<br /><strong>（1）每个节点或者是黑色，或者是红色。</strong><br /><strong>（2）根节点是黑色。</strong><br /><strong>（3）每个叶子节点（NIL）是黑色。&nbsp;[注意：这里叶子节点，是指为空(NIL或NULL)的叶子节点！]</strong><br /><strong>（4）如果一个节点是红色的，则它的子节点必须是黑色的。</strong><br /><strong>（5）从一个节点到该节点的子孙节点的所有路径上包含相同数目的黑节点。</strong></p>
<p><strong>注意</strong>：<br />(01) 特性(3)中的叶子节点，是只为空(NIL或null)的节点。<br />(02) 特性(5)，确保没有一条路径会比其他路径长出俩倍。因而，红黑树是相对是接近平衡的二叉树。</p>

<h3><strong><a name="aa2"></a>红黑树的应用</strong></h3>
<p>红黑树的应用比较广泛，主要是用它来存储有序的数据，它的时间复杂度是O(lgn)，效率非常之高。<br />例如，Java集合中的<a href="http://www.cnblogs.com/skywang12345/p/3311268.html">TreeSet</a>和<a href="http://www.cnblogs.com/skywang12345/p/3310928.html">TreeMap</a>，C++ STL中的set、map，以及Linux虚拟内存的管理，都是通过红黑树去实现的。</p>
<p>&nbsp;</p>
<h3><a name="a2"></a><strong>红黑树的时间复杂度和相关证明<br /></strong></h3>
<p><strong>红黑树的时间复杂度为: O(lgn)</strong><br />下面通过&ldquo;<em>数学归纳法</em>&rdquo;对红黑树的时间复杂度进行证明。</p>
<p>定理：<strong>一棵含有n个节点的红黑树的高度至多为2log(n+1)</strong>.</p>
<p>证明：<br />&nbsp; &nbsp; "一棵含有n个节点的红黑树的高度至多为2log(n+1)" 的<strong>逆否命题</strong>是 "高度为h的红黑树，它的包含的内节点个数至少为 2<sup>h/2</sup>-1个"。<br />&nbsp; &nbsp; 我们只需要证明逆否命题，即可证明原命题为真；即只需证明&nbsp;"高度为h的红黑树，它的包含的内节点个数至少为 2<sup>h/2</sup>-1个"。</p>
<p>&nbsp; &nbsp; 从某个节点x出发（不包括该节点）到达一个叶节点的任意一条路径上，黑色节点的个数称为该节点的黑高度(x's black height)，记为<strong>bh(x)</strong>。关于bh(x)有两点需要说明：&nbsp;<br />&nbsp; &nbsp; 第1点：根据红黑树的"<strong>特性(5)</strong>&nbsp;，即<em>从一个节点到该节点的子孙节点的所有路径上包含相同数目的黑节点</em>"可知，从节点x出发到达的所有的叶节点具有相同数目的黑节点。<strong>这也就意味着，bh(x)的值是唯一的</strong>！<br />&nbsp; &nbsp; 第2点：根据红黑色的"特性(4)，即<em>如果一个节点是红色的，则它的子节点必须是黑色的</em>"可知，从节点x出发达到叶节点"所经历的黑节点数目"&gt;= "所经历的红节点的数目"。假设x是根节点，则可以得出结论"<strong>bh(x) &gt;= h/2</strong>"。进而，我们只需证明 "高度为h的红黑树，它的包含的黑节点个数至少为 2<sup>bh(x)</sup>-1个"即可。</p>
<p>&nbsp; &nbsp; 到这里，我们将需要证明的定理已经由<br /><strong>"一棵含有n个节点的红黑树的高度至多为2log(n+1)"</strong>&nbsp;<br />&nbsp; &nbsp; 转变成只需要证明<br /><strong>"高度为h的红黑树，它的包含的内节点个数至少为 2<sup>bh(x)</sup>-1个"。</strong></p>
<p><br />下面通过"数学归纳法"开始论证高度为h的红黑树，它的包含的内节点个数至少为 2<sup>bh(x)</sup>-1个"。</p>
<p>(01) 当树的高度h=0时，<br />&nbsp; &nbsp; 内节点个数是0，bh(x) 为0，2<sup>bh(x)</sup>-1 也为 0。显然，原命题成立。</p>
<p>(02) 当h&gt;0，且树的高度为 h-1 时，它包含的节点个数至少为 2<sup>bh(x)-1</sup>-1。这个是根据(01)推断出来的！</p>
<p>&nbsp; &nbsp; 下面，由树的高度为 h-1 的已知条件推出&ldquo;树的高度为 h 时，它所包含的节点树为 2<sup>bh(x)</sup>-1&rdquo;。</p>
<p>&nbsp; &nbsp; 当树的高度为 h 时，<br />&nbsp; &nbsp; 对于节点x(x为根节点)，其黑高度为bh(x)。<br />&nbsp; &nbsp; 对于节点x的左右子树，它们黑高度为 bh(x) 或者 bh(x)-1。<br />&nbsp; &nbsp; 根据(02)的已知条件，我们已知 "x的左右子树，即高度为 h-1 的节点，它包含的节点至少为 2<sup>bh(x)-1</sup>-1 个"；</p>
<p>&nbsp; &nbsp; 所以，节点x所包含的节点至少为 ( 2<sup>bh(x)-1</sup>-1 ) + ( 2<sup>bh(x)-1</sup>-1 ) + 1 = 2^<sup>bh(x)</sup>-1。即节点x所包含的节点至少为 2<sup>bh(x)</sup>-1。<br />&nbsp; &nbsp; 因此，原命题成立。</p>
<p>&nbsp; &nbsp; 由(01)、(02)得出，"高度为h的红黑树，它的包含的内节点个数至少为 2^<sup>bh(x)</sup>-1个"。<br />&nbsp; &nbsp; 因此，&ldquo;一棵含有n个节点的红黑树的高度至多为2log(n+1)&rdquo;。</p>

<h3><a name="a3"></a><strong>红黑树的基本操作(一) 左旋和右旋</strong></h3>
<p>红黑树的基本操作是<strong>添加</strong>、<strong>删除</strong>。在对红黑树进行添加或删除之后，都会用到旋转方法。为什么呢？道理很简单，添加或删除红黑树中的节点之后，红黑树就发生了变化，可能不满足红黑树的5条性质，也就不再是一颗红黑树了，而是一颗普通的树。而通过旋转，可以使这颗树重新成为红黑树。简单点说，旋转的目的是让树保持红黑树的特性。<br />旋转包括两种：<strong>左旋</strong>&nbsp;和&nbsp;<strong>右旋</strong>。下面分别对它们进行介绍。</p>

<h3><strong><a name="aa5"></a>红黑树的基本操作(二) 添加</strong></h3>
<p>将一个节点插入到红黑树中，需要执行哪些步骤呢？首先，将红黑树当作一颗二叉查找树，将节点插入；然后，将节点着色为红色；最后，通过旋转和重新着色等方法来修正该树，使之重新成为一颗红黑树。详细描述如下：</p>
<p><strong>第一步: 将红黑树当作一颗二叉查找树，将节点插入。</strong><br />&nbsp; &nbsp; &nbsp; &nbsp;红黑树本身就是一颗二叉查找树，将节点插入后，该树仍然是一颗二叉查找树。也就意味着，树的键值仍然是有序的。此外，无论是左旋还是右旋，若旋转之前这棵树是二叉查找树，旋转之后它一定还是二叉查找树。这也就意味着，任何的旋转和重新着色操作，都不会改变它仍然是一颗二叉查找树的事实。<br />&nbsp; &nbsp; &nbsp; &nbsp;好吧？那接下来，我们就来想方设法的旋转以及重新着色，使这颗树重新成为红黑树！</p>
<p><strong>第二步：将插入的节点着色为"红色"。</strong><br />&nbsp; &nbsp; &nbsp; &nbsp;为什么着色成红色，而不是黑色呢？为什么呢？在回答之前，我们需要重新温习一下红黑树的特性：<br />(1) 每个节点或者是黑色，或者是红色。<br />(2) 根节点是黑色。<br />(3) 每个叶子节点是黑色。 [注意：这里叶子节点，是指为空的叶子节点！]<br />(4) 如果一个节点是红色的，则它的子节点必须是黑色的。<br />(5) 从一个节点到该节点的子孙节点的所有路径上包含相同数目的黑节点。<br />&nbsp; &nbsp; &nbsp; &nbsp;将插入的节点着色为红色，不会违背"特性(5)"！少违背一条特性，就意味着我们需要处理的情况越少。接下来，就要努力的让这棵树满足其它性质即可；满足了的话，它就又是一颗红黑树了。o(&cap;&cap;)o...哈哈</p>
<p><strong>第三步: 通过一系列的旋转或着色等操作，使之重新成为一颗红黑树。</strong><br />&nbsp; &nbsp; &nbsp; &nbsp;第二步中，将插入节点着色为"红色"之后，不会违背"特性(5)"。那它到底会违背哪些特性呢？<br />&nbsp; &nbsp; &nbsp; &nbsp;对于"特性(1)"，显然不会违背了。因为我们已经将它涂成红色了。<br />&nbsp; &nbsp; &nbsp; &nbsp;对于"特性(2)"，显然也不会违背。在第一步中，我们是将红黑树当作二叉查找树，然后执行的插入操作。而根据二叉查找数的特点，插入操作不会改变根节点。所以，根节点仍然是黑色。<br />&nbsp; &nbsp; &nbsp; &nbsp;对于"特性(3)"，显然不会违背了。这里的叶子节点是指的空叶子节点，插入非空节点并不会对它们造成影响。<br />&nbsp; &nbsp; &nbsp; &nbsp;对于"特性(4)"，是有可能违背的！<br />&nbsp; &nbsp; &nbsp; &nbsp;那接下来，想办法使之"满足特性(4)"，就可以将树重新构造成红黑树了。</p>

<p>上面三种情况(Case)处理问题的核心思路都是：将红色的节点移到根节点；然后，将根节点设为黑色。下面对它们详细进行介绍。</p>
<p>&nbsp;</p>
<p><strong>1. (Case 1)叔叔是红色</strong></p>
<p><strong>1.1 现象说明</strong><br />当前节点(即，被插入节点)的父节点是红色，且当前节点的祖父节点的另一个子节点（叔叔节点）也是红色。</p>
<p><strong>1.2 处理策略</strong><br />(01) 将&ldquo;父节点&rdquo;设为黑色。<br />(02) 将&ldquo;叔叔节点&rdquo;设为黑色。<br />(03) 将&ldquo;祖父节点&rdquo;设为&ldquo;红色&rdquo;。<br />(04) 将&ldquo;祖父节点&rdquo;设为&ldquo;当前节点&rdquo;(红色节点)；即，之后继续对&ldquo;当前节点&rdquo;进行操作。</p>
<p>&nbsp; &nbsp;&nbsp;<strong>下面谈谈为什么要这样处理。</strong>(建议理解的时候，通过下面的图进行对比)<br />&nbsp; &nbsp; &ldquo;当前节点&rdquo;和&ldquo;父节点&rdquo;都是红色，违背&ldquo;特性(4)&rdquo;。所以，将&ldquo;父节点&rdquo;设置&ldquo;黑色&rdquo;以解决这个问题。<br />&nbsp; &nbsp; 但是，将&ldquo;父节点&rdquo;由&ldquo;红色&rdquo;变成&ldquo;黑色&rdquo;之后，违背了&ldquo;特性(5)&rdquo;：因为，包含&ldquo;父节点&rdquo;的分支的黑色节点的总数增加了1。 &nbsp;解决这个问题的办法是：将&ldquo;祖父节点&rdquo;由&ldquo;黑色&rdquo;变成红色，同时，将&ldquo;叔叔节点&rdquo;由&ldquo;红色&rdquo;变成&ldquo;黑色&rdquo;。关于这里，说明几点：第一，为什么&ldquo;祖父节点&rdquo;之前是黑色？这个应该很容易想明白，因为在变换操作之前，该树是红黑树，&ldquo;父节点&rdquo;是红色，那么&ldquo;祖父节点&rdquo;一定是黑色。 第二，为什么将&ldquo;祖父节点&rdquo;由&ldquo;黑色&rdquo;变成红色，同时，将&ldquo;叔叔节点&rdquo;由&ldquo;红色&rdquo;变成&ldquo;黑色&rdquo;；能解决&ldquo;包含&lsquo;父节点&rsquo;的分支的黑色节点的总数增加了1&rdquo;的问题。这个道理也很简单。&ldquo;包含&lsquo;父节点&rsquo;的分支的黑色节点的总数增加了1&rdquo; 同时也意味着 &ldquo;包含&lsquo;祖父节点&rsquo;的分支的黑色节点的总数增加了1&rdquo;，既然这样，我们通过将&ldquo;祖父节点&rdquo;由&ldquo;黑色&rdquo;变成&ldquo;红色&rdquo;以解决&ldquo;包含&lsquo;祖父节点&rsquo;的分支的黑色节点的总数增加了1&rdquo;的问题； 但是，这样处理之后又会引起另一个问题&ldquo;包含&lsquo;叔叔&rsquo;节点的分支的黑色节点的总数减少了1&rdquo;，现在我们已知&ldquo;叔叔节点&rdquo;是&ldquo;红色&rdquo;，将&ldquo;叔叔节点&rdquo;设为&ldquo;黑色&rdquo;就能解决这个问题。 所以，将&ldquo;祖父节点&rdquo;由&ldquo;黑色&rdquo;变成红色，同时，将&ldquo;叔叔节点&rdquo;由&ldquo;红色&rdquo;变成&ldquo;黑色&rdquo;；就解决了该问题。<br />&nbsp; &nbsp; 按照上面的步骤处理之后：当前节点、父节点、叔叔节点之间都不会违背红黑树特性，但祖父节点却不一定。若此时，祖父节点是根节点，直接将祖父节点设为&ldquo;黑色&rdquo;，那就完全解决这个问题了；若祖父节点不是根节点，那我们需要将&ldquo;祖父节点&rdquo;设为&ldquo;新的当前节点&rdquo;，接着对&ldquo;新的当前节点&rdquo;进行分析。</p>

<p><strong>2. (Case 2)叔叔是黑色，且当前节点是右孩子</strong></p>
<p><strong>2.1 现象说明</strong><br />当前节点(即，被插入节点)的父节点是红色，叔叔节点是黑色，且当前节点是其父节点的右孩子</p>
<p><strong>2.2 处理策略</strong><br />(01) 将&ldquo;父节点&rdquo;作为&ldquo;新的当前节点&rdquo;。<br />(02) 以&ldquo;新的当前节点&rdquo;为支点进行左旋。</p>
<p>&nbsp; &nbsp; &nbsp;&nbsp;<strong>下面谈谈为什么要这样处理。</strong>(建议理解的时候，通过下面的图进行对比)<br />&nbsp; &nbsp; &nbsp; 首先，将&ldquo;父节点&rdquo;作为&ldquo;新的当前节点&rdquo;；接着，以&ldquo;新的当前节点&rdquo;为支点进行左旋。 为了便于理解，我们先说明第(02)步，再说明第(01)步；为了便于说明，我们设置&ldquo;父节点&rdquo;的代号为F(Father)，&ldquo;当前节点&rdquo;的代号为S(Son)。<br />为什么要&ldquo;以F为支点进行左旋&rdquo;呢？根据已知条件可知：S是F的右孩子。而之前我们说过，我们处理红黑树的核心思想：将红色的节点移到根节点；然后，将根节点设为黑色。既然是&ldquo;将红色的节点移到根节点&rdquo;，那就是说要不断的将破坏红黑树特性的红色节点上移(即向根方向移动)。 而S又是一个右孩子，因此，我们可以通过&ldquo;左旋&rdquo;来将S上移！&nbsp;<br />&nbsp; &nbsp; &nbsp; 按照上面的步骤(以F为支点进行左旋)处理之后：若S变成了根节点，那么直接将其设为&ldquo;黑色&rdquo;，就完全解决问题了；若S不是根节点，那我们需要执行步骤(01)，即&ldquo;将F设为&lsquo;新的当前节点&rsquo;&rdquo;。那为什么不继续以S为新的当前节点继续处理，而需要以F为新的当前节点来进行处理呢？这是因为&ldquo;左旋&rdquo;之后，F变成了S的&ldquo;子节点&rdquo;，即S变成了F的父节点；而我们处理问题的时候，需要从下至上(由叶到根)方向进行处理；也就是说，必须先解决&ldquo;孩子&rdquo;的问题，再解决&ldquo;父亲&rdquo;的问题；所以，我们执行步骤(01)：将&ldquo;父节点&rdquo;作为&ldquo;新的当前节点&rdquo;。</p>

<p>&nbsp; &nbsp; &nbsp;&nbsp;<strong>下面谈谈为什么要这样处理。</strong>(建议理解的时候，通过下面的图进行对比)<br />&nbsp; &nbsp; &nbsp; 为了便于说明，我们设置&ldquo;当前节点&rdquo;为S(Original Son)，&ldquo;兄弟节点&rdquo;为B(Brother)，&ldquo;叔叔节点&rdquo;为U(Uncle)，&ldquo;父节点&rdquo;为F(Father)，祖父节点为G(Grand-Father)。<br />&nbsp; &nbsp; &nbsp; S和F都是红色，违背了红黑树的&ldquo;特性(4)&rdquo;，我们可以将F由&ldquo;红色&rdquo;变为&ldquo;黑色&rdquo;，就解决了&ldquo;违背&lsquo;特性(4)&rsquo;&rdquo;的问题；但却引起了其它问题：违背特性(5)，因为将F由红色改为黑色之后，所有经过F的分支的黑色节点的个数增加了1。那我们如何解决&ldquo;所有经过F的分支的黑色节点的个数增加了1&rdquo;的问题呢？ 我们可以通过&ldquo;将G由黑色变成红色&rdquo;，同时&ldquo;以G为支点进行右旋&rdquo;来解决。</p>

<h3><a name="a35"></a><strong>红黑树的基本操作(三) 删除</strong></h3>
<p>将红黑树内的某一个节点删除。需要执行的操作依次是：首先，将红黑树当作一颗二叉查找树，将该节点从二叉查找树中删除；然后，通过"旋转和重新着色"等一系列来修正该树，使之重新成为一棵红黑树。详细描述如下：</p>
<p><strong>第一步：将红黑树当作一颗二叉查找树，将节点删除。</strong><br />&nbsp; &nbsp; &nbsp; &nbsp;这和"删除常规二叉查找树中删除节点的方法是一样的"。分3种情况：<br />&nbsp; &nbsp; &nbsp; &nbsp;① 被删除节点没有儿子，即为叶节点。那么，直接将该节点删除就OK了。<br />&nbsp; &nbsp; &nbsp; &nbsp;② 被删除节点只有一个儿子。那么，直接删除该节点，并用该节点的唯一子节点顶替它的位置。<br />&nbsp; &nbsp; &nbsp; &nbsp;③ 被删除节点有两个儿子。那么，先找出它的后继节点；然后把&ldquo;它的后继节点的内容&rdquo;复制给&ldquo;该节点的内容&rdquo;；之后，删除&ldquo;它的后继节点&rdquo;。在这里，后继节点相当于替身，在将后继节点的内容复制给"被删除节点"之后，再将后继节点删除。这样就巧妙的将问题转换为"删除后继节点"的情况了，下面就考虑后继节点。 在"被删除节点"有两个非空子节点的情况下，它的后继节点不可能是双子非空。既然"的后继节点"不可能双子都非空，就意味着"该节点的后继节点"要么没有儿子，要么只有一个儿子。若没有儿子，则按"情况① "进行处理；若只有一个儿子，则按"情况② "进行处理。</p>
<p><strong>第二步：通过"旋转和重新着色"等一系列来修正该树，使之重新成为一棵红黑树。</strong><br />&nbsp; &nbsp; &nbsp; &nbsp;因为"第一步"中删除节点之后，可能会违背红黑树的特性。所以需要通过"旋转和重新着色"来修正该树，使之重新成为一棵红黑树。</p>

<p>下面对删除函数进行分析。在分析之前，我们再次温习一下红黑树的几个特性：<br />(1) 每个节点或者是黑色，或者是红色。<br />(2) 根节点是黑色。<br />(3) 每个叶子节点是黑色。 [注意：这里叶子节点，是指为空的叶子节点！]<br />(4) 如果一个节点是红色的，则它的子节点必须是黑色的。<br />(5) 从一个节点到该节点的子孙节点的所有路径上包含相同数目的黑节点。</p>
<p><br />&nbsp; &nbsp; &nbsp; 前面我们将"删除红黑树中的节点"大致分为两步，在第一步中"将红黑树当作一颗二叉查找树，将节点删除"后，可能违反"特性(2)、(4)、(5)"三个特性。第二步需要解决上面的三个问题，进而保持红黑树的全部特性。<br />&nbsp; &nbsp; &nbsp; 为了便于分析，我们假设"x包含一个额外的黑色"(x原本的颜色还存在)，这样就不会违反"特性(5)"。为什么呢？<br />&nbsp; &nbsp; &nbsp; 通过RB-DELETE算法，我们知道：删除节点y之后，x占据了原来节点y的位置。 既然删除y(y是黑色)，意味着减少一个黑色节点；那么，再在该位置上增加一个黑色即可。这样，当我们假设"x包含一个额外的黑色"，就正好弥补了"删除y所丢失的黑色节点"，也就不会违反"特性(5)"。 因此，假设"x包含一个额外的黑色"(x原本的颜色还存在)，这样就不会违反"特性(5)"。<br />&nbsp; &nbsp; &nbsp; 现在，x不仅包含它原本的颜色属性，x还包含一个额外的黑色。即x的颜色属性是"红+黑"或"黑+黑"，它违反了"特性(1)"。</p>
<p>&nbsp; &nbsp; &nbsp; 现在，我们面临的问题，由解决"违反了特性(2)、(4)、(5)三个特性"转换成了"解决违反特性(1)、(2)、(4)三个特性"。RB-DELETE-FIXUP需要做的就是通过算法恢复红黑树的特性(1)、(2)、(4)。RB-DELETE-FIXUP的思想是：将x所包含的额外的黑色不断沿树上移(向根方向移动)，直到出现下面的姿态：<br />a) x指向一个"红+黑"节点。此时，将x设为一个"黑"节点即可。<br />b) x指向根。此时，将x设为一个"黑"节点即可。<br />c) 非前面两种姿态。</p>
<p>将上面的姿态，可以概括为3种情况。<br />① 情况说明：x是&ldquo;红+黑&rdquo;节点。<br />&nbsp; &nbsp; 处理方法：直接把x设为黑色，结束。此时红黑树性质全部恢复。<br />② 情况说明：x是&ldquo;黑+黑&rdquo;节点，且x是根。<br />&nbsp; &nbsp; 处理方法：什么都不做，结束。此时红黑树性质全部恢复。<br />③ 情况说明：x是&ldquo;黑+黑&rdquo;节点，且x不是根。<br />&nbsp; &nbsp; 处理方法：这种情况又可以划分为4种子情况。这4种子情况如下表所示：</p>

<p><strong>1. (Case 1)x是"黑+黑"节点，x的兄弟节点是红色</strong></p>
<p><strong>1.1 现象说明</strong><br />x是"黑+黑"节点，x的兄弟节点是红色。(此时x的父节点和x的兄弟节点的子节点都是黑节点)。</p>
<p><strong>1.2 处理策略</strong><br />(01) 将x的兄弟节点设为&ldquo;黑色&rdquo;。<br />(02) 将x的父节点设为&ldquo;红色&rdquo;。<br />(03) 对x的父节点进行左旋。<br />(04) 左旋后，重新设置x的兄弟节点。</p>
<p>&nbsp; &nbsp; &nbsp;&nbsp;<strong>下面谈谈为什么要这样处理。</strong>(建议理解的时候，通过下面的图进行对比)<br />&nbsp; &nbsp; &nbsp; 这样做的目的是将&ldquo;Case 1&rdquo;转换为&ldquo;Case 2&rdquo;、&ldquo;Case 3&rdquo;或&ldquo;Case 4&rdquo;，从而进行进一步的处理。对x的父节点进行左旋；左旋后，为了保持红黑树特性，就需要在左旋前&ldquo;将x的兄弟节点设为黑色&rdquo;，同时&ldquo;将x的父节点设为红色&rdquo;；左旋后，由于x的兄弟节点发生了变化，需要更新x的兄弟节点，从而进行后续处理。</p>

<p><strong>2. (Case 2) x是"黑+黑"节点，x的兄弟节点是黑色，x的兄弟节点的两个孩子都是黑色</strong></p>
<p><strong>2.1 现象说明</strong><br />x是&ldquo;黑+黑&rdquo;节点，x的兄弟节点是黑色，x的兄弟节点的两个孩子都是黑色。</p>
<p><strong>2.2 处理策略</strong><br />(01) 将x的兄弟节点设为&ldquo;红色&rdquo;。<br />(02) 设置&ldquo;x的父节点&rdquo;为&ldquo;新的x节点&rdquo;。</p>
<p>&nbsp; &nbsp; &nbsp;&nbsp;<strong>下面谈谈为什么要这样处理。</strong>(建议理解的时候，通过下面的图进行对比)<br />&nbsp; &nbsp; &nbsp; 这个情况的处理思想：是将&ldquo;x中多余的一个黑色属性上移(往根方向移动)&rdquo;。 x是&ldquo;黑+黑&rdquo;节点，我们将x由&ldquo;黑+黑&rdquo;节点 变成 &ldquo;黑&rdquo;节点，多余的一个&ldquo;黑&rdquo;属性移到x的父节点中，即x的父节点多出了一个黑属性(若x的父节点原先是&ldquo;黑&rdquo;，则此时变成了&ldquo;黑+黑&rdquo;；若x的父节点原先时&ldquo;红&rdquo;，则此时变成了&ldquo;红+黑&rdquo;)。 此时，需要注意的是：所有经过x的分支中黑节点个数没变化；但是，所有经过x的兄弟节点的分支中黑色节点的个数增加了1(因为x的父节点多了一个黑色属性)！为了解决这个问题，我们需要将&ldquo;所有经过x的兄弟节点的分支中黑色节点的个数减1&rdquo;即可，那么就可以通过&ldquo;将x的兄弟节点由黑色变成红色&rdquo;来实现。<br />&nbsp; &nbsp; &nbsp; 经过上面的步骤(将x的兄弟节点设为红色)，多余的一个颜色属性(黑色)已经跑到x的父节点中。我们需要将x的父节点设为&ldquo;新的x节点&rdquo;进行处理。若&ldquo;新的x节点&rdquo;是&ldquo;黑+红&rdquo;，直接将&ldquo;新的x节点&rdquo;设为黑色，即可完全解决该问题；若&ldquo;新的x节点&rdquo;是&ldquo;黑+黑&rdquo;，则需要对&ldquo;新的x节点&rdquo;进行进一步处理。</p>

<p><strong>3. (Case 3)x是&ldquo;黑+黑&rdquo;节点，x的兄弟节点是黑色；x的兄弟节点的左孩子是红色，右孩子是黑色的</strong></p>
<p><strong>3.1 现象说明</strong><br />x是&ldquo;黑+黑&rdquo;节点，x的兄弟节点是黑色；x的兄弟节点的左孩子是红色，右孩子是黑色的。</p>
<p><strong>3.2 处理策略</strong><br />(01) 将x兄弟节点的左孩子设为&ldquo;黑色&rdquo;。<br />(02) 将x兄弟节点设为&ldquo;红色&rdquo;。<br />(03) 对x的兄弟节点进行右旋。<br />(04) 右旋后，重新设置x的兄弟节点。</p>
<p>&nbsp; &nbsp; &nbsp; &nbsp;<strong>下面谈谈为什么要这样处理。</strong>(建议理解的时候，通过下面的图进行对比)<br />&nbsp; &nbsp; &nbsp; &nbsp;我们处理&ldquo;Case 3&rdquo;的目的是为了将&ldquo;Case 3&rdquo;进行转换，转换成&ldquo;Case 4&rdquo;,从而进行进一步的处理。转换的方式是对x的兄弟节点进行右旋；为了保证右旋后，它仍然是红黑树，就需要在右旋前&ldquo;将x的兄弟节点的左孩子设为黑色&rdquo;，同时&ldquo;将x的兄弟节点设为红色&rdquo;；右旋后，由于x的兄弟节点发生了变化，需要更新x的兄弟节点，从而进行后续处理。</p>

<p>&nbsp;</p>
<p><strong>4. (Case 4)x是&ldquo;黑+黑&rdquo;节点，x的兄弟节点是黑色；x的兄弟节点的右孩子是红色的，x的兄弟节点的左孩子任意颜色</strong></p>
<p><strong>4.1 现象说明</strong><br />x是&ldquo;黑+黑&rdquo;节点，x的兄弟节点是黑色；x的兄弟节点的右孩子是红色的，x的兄弟节点的左孩子任意颜色。</p>
<p><strong>4.2 处理策略</strong><br />(01) 将x父节点颜色 赋值给 x的兄弟节点。<br />(02) 将x父节点设为&ldquo;黑色&rdquo;。<br />(03) 将x兄弟节点的右子节设为&ldquo;黑色&rdquo;。<br />(04) 对x的父节点进行左旋。<br />(05) 设置&ldquo;x&rdquo;为&ldquo;根节点&rdquo;。</p>
<p>&nbsp; &nbsp; &nbsp;&nbsp;<strong>下面谈谈为什么要这样处理。</strong>(建议理解的时候，通过下面的图进行对比)<br />&nbsp; &nbsp; &nbsp; 我们处理&ldquo;Case 4&rdquo;的目的是：去掉x中额外的黑色，将x变成单独的黑色。处理的方式是&ldquo;：进行颜色修改，然后对x的父节点进行左旋。下面，我们来分析是如何实现的。<br />&nbsp; &nbsp; &nbsp; 为了便于说明，我们设置&ldquo;当前节点&rdquo;为S(Original Son)，&ldquo;兄弟节点&rdquo;为B(Brother)，&ldquo;兄弟节点的左孩子&rdquo;为BLS(Brother's Left Son)，&ldquo;兄弟节点的右孩子&rdquo;为BRS(Brother's Right Son)，&ldquo;父节点&rdquo;为F(Father)。<br />&nbsp; &nbsp; &nbsp; 我们要对F进行左旋。但在左旋前，我们需要调换F和B的颜色，并设置BRS为黑色。为什么需要这里处理呢？因为左旋后，F和BLS是父子关系，而我们已知BL是红色，如果F是红色，则违背了&ldquo;特性(4)&rdquo;；为了解决这一问题，我们将&ldquo;F设置为黑色&rdquo;。 但是，F设置为黑色之后，为了保证满足&ldquo;特性(5)&rdquo;，即为了保证左旋之后：<br />&nbsp; &nbsp; &nbsp; 第一，&ldquo;同时经过根节点和S的分支的黑色节点个数不变&rdquo;。<br />&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;若满足&ldquo;第一&rdquo;，只需要S丢弃它多余的颜色即可。因为S的颜色是&ldquo;黑+黑&rdquo;，而左旋后&ldquo;同时经过根节点和S的分支的黑色节点个数&rdquo;增加了1；现在，只需将S由&ldquo;黑+黑&rdquo;变成单独的&ldquo;黑&rdquo;节点，即可满足&ldquo;第一&rdquo;。<br />&nbsp; &nbsp; &nbsp; 第二，&ldquo;同时经过根节点和BLS的分支的黑色节点数不变&rdquo;。<br />&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;若满足&ldquo;第二&rdquo;，只需要将&ldquo;F的原始颜色&rdquo;赋值给B即可。之前，我们已经将&ldquo;F设置为黑色&rdquo;(即，将B的颜色"黑色"，赋值给了F)。至此，我们算是调换了F和B的颜色。<br />&nbsp; &nbsp; &nbsp; 第三，&ldquo;同时经过根节点和BRS的分支的黑色节点数不变&rdquo;。<br />&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;在&ldquo;第二&rdquo;已经满足的情况下，若要满足&ldquo;第三&rdquo;，只需要将BRS设置为&ldquo;黑色&rdquo;即可。<br />经过，上面的处理之后。红黑树的特性全部得到的满足！接着，我们将x设为根节点，就可以跳出while循环(参考伪代码)；即完成了全部处理。</p>
<p>至此，我们就完成了Case 4的处理。理解Case 4的核心，是了解如何&ldquo;去掉当前节点额外的黑色&rdquo;。</p>
